# Prompt Cheat Sheet

> Web Access Skill — 所有可用的 Prompt 触发词和能力速查
> 关联教程：[explorer/02-web-access-guide.md](explorer/02-web-access-guide.md)

## 一、三层通道调度

### 1. 直接访问（Direct Fetch）

| 字段 | 内容 |
|------|------|
| 触发场景 | URL 已知 + 需要定向提取特定信息 |
| 说明 | 拉取网页内容，由小模型根据 prompt 提取，返回处理后结果 |
| 涉及脚本 | 无 |
| 示例 Prompt | "读取 https://example.com 这篇文章的核心观点"<br>"提取 https://example.com 中的价格信息"<br>"总结这个页面的内容：[URL]" |

### 2. 浏览器访问（Browser via CDP）

| 字段 | 内容 |
|------|------|
| 触发场景 | 非公开内容 / 需要登录态 / 需要交互操作 / 已知反爬平台 / 需要自由导航探索 |
| 说明 | 直连用户日常 Chrome，天然携带登录态，通过 CDP 操作真实浏览器 |
| 涉及脚本 | `check-deps.mjs`, `cdp-proxy.mjs`, `match-site.mjs` |
| 示例 Prompt | "登录我的推特账户并查看最新消息"<br>"在小红书上搜索[关键词]并提取前10个结果"<br>"帮我操作[网站]完成下单流程" |

### 3. 搜索发现

| 字段 | 内容 |
|------|------|
| 触发场景 | 搜索摘要或关键词结果，发现信息来源 |
| 说明 | 用于信息发现入口，定位信息来源后升级到直接访问 |
| 涉及脚本 | 无 |
| 示例 Prompt | "搜索最新的 AI 新闻"<br>"查找关于[主题]的资料" |

### 4. 本地检索（Local Chrome Search）

| 字段 | 内容 |
|------|------|
| 触发场景 | 用户指向"本人访问过的页面" / 组织内部系统 |
| 说明 | 检索本地 Chrome 书签/历史记录 |
| 涉及脚本 | `find-url.mjs` |
| 示例 Prompt | "我之前看的那个讲 React 的文章在哪"<br>"找回我上周打开的 GitHub 面板"<br>"查找我们公司的内部系统地址" |

## 二、CDP 浏览器操作

### 页面导航

| 字段 | 内容 |
|------|------|
| 触发场景 | 需要在页面内导航 / 打开新链接 |
| 说明 | 通过 `/click` 在当前 tab 操作，或 `/new` + 完整 URL 打开新 tab |
| 示例 Prompt | "点击这个链接进入详情页"<br>"在新标签页打开所有搜索结果"<br>"返回上一页"<br>"刷新页面" |

### DOM 操作与提取

| 字段 | 内容 |
|------|------|
| 触发场景 | 需要查看页面结构 / 提取页面内容 / 读取折叠、懒加载内容 |
| 说明 | 通过 `/eval` 查询 DOM，穿透 Shadow DOM、iframe 等边界 |
| 示例 Prompt | "看看这个页面有什么内容"<br>"提取所有文章标题和链接"<br>"获取折叠区块里的文字"<br>"列出页面上所有图片的 URL" |

### 交互操作

| 字段 | 内容 |
|------|------|
| 触发场景 | 需要点击按钮 / 填写表单 / 滚动加载 / 上传文件 |
| 说明 | 通过 `/click`、`/scroll`、`/eval` 等进行 GUI 交互 |
| 示例 Prompt | "点击展开按钮"<br>"滚动到底部加载更多"<br>"填写搜索框：[关键词]"<br>"上传文件 /path/to/file.png" |

### 媒体资源提取

| 字段 | 内容 |
|------|------|
| 触发场景 | 需要提取图片/视频 |
| 说明 | 通过 `/eval` 从 DOM 直接拿媒体 URL，再定向读取 |
| 示例 Prompt | "提取页面中所有高清图片"<br>"下载这个视频的封面图"<br>"获取轮播图中所有图片" |

### 视频内容分析

| 字段 | 内容 |
|------|------|
| 触发场景 | 需要分析视频内容 / 提取视频关键帧 |
| 说明 | 通过 `/eval` 操控 `<video>` 元素，配合 `/screenshot` 采帧 |
| 示例 Prompt | "分析这个视频讲了什么"<br>"提取视频的关键画面"<br>"跳到视频 30 秒处截图" |

## 三、辅助能力

### 信息核实

| 字段 | 内容 |
|------|------|
| 触发场景 | 需要核实信息真伪 / 查找一手来源 |
| 说明 | 以一手来源为核，避免循环印证假象 |
| 示例 Prompt | "核实这条新闻的真实性"<br>"查找[政策]的官方原文" |

### 并行调研（子 Agent 分治）

| 字段 | 内容 |
|------|------|
| 触发场景 | 多个独立调研目标 / 需要同时抓取多个页面 |
| 说明 | 分治给子 Agent 并行执行，节省时间并保护主上下文 |
| 示例 Prompt | "同时调研这 5 个竞品网站"<br>"批量抓取 10 个页面的数据"<br>"对比分析 3 个不同来源的信息" |

### 站点经验应用

| 字段 | 内容 |
|------|------|
| 触发场景 | 访问已知平台 / 需要避免已知的反爬陷阱 |
| 说明 | 自动读取 `references/site-patterns/{domain}.md` 获取先验知识 |
| 触发方式 | 自动触发，无需显式 prompt |

### Jina 预处理（可选）

| 字段 | 内容 |
|------|------|
| 触发场景 | 需要节省 tokens / 页面是文章/博客/文档类 |
| 说明 | 使用 `r.jina.ai/` 前缀将网页转为 Markdown，限 20 RPM |
| 示例 Prompt | "用 Jina 读取这篇文章节省 tokens"<br>"提取这个博客的正文内容（用 Jina）" |

## 四、完整 Prompt 速查表

| 想做什么 | 可以这样问 | 使用的通道 |
|---------|-----------|-----------|
| **搜索信息** | "搜索[主题]" / "查找关于[关键词]的资料" | WebSearch |
| **读取已知 URL** | "读取 https://example.com" / "提取这个页面的[内容]：[URL]" | WebFetch / Jina |
| **需要登录的操作** | "登录[网站]并查看[内容]" / "在我已登录的状态下操作[网站]" | CDP 浏览器 |
| **抓取反爬平台** | "抓取小红书关于[主题]的内容" / "获取微信公众号文章：[URL]" | CDP 浏览器 |
| **操作网页界面** | "点击[按钮]，填写[表单]，提交" / "在[网站]上完成[操作]流程" | CDP 浏览器 |
| **提取媒体资源** | "提取页面中所有图片链接" / "下载这个视频" | CDP + DOM |
| **分析视频内容** | "分析这个视频讲了什么" / "提取视频关键帧" | CDP + video 操控 |
| **找回历史页面** | "我之前看的那个[主题]文章在哪" / "搜索我上周打开的[网站]" | 本地 Chrome 检索 |
| **核实信息** | "核实这条新闻的真实性" / "查找[政策]的官方原文" | WebSearch → 一手来源 |
| **批量调研** | "同时调研这 N 个[目标]" / "对比分析这 3 个网站" | 并行子 Agent + CDP |
| **提取折叠/懒加载内容** | "展开所有折叠区块并提取内容" / "滚动加载并获取所有数据" | CDP + scroll + eval |
| **上传文件** | "在[网站]上传文件 /path/to/file.png" | CDP + setFiles |
| **组织内部系统** | "打开我们的[内部系统]" / "访问公司的[平台]" | 本地 Chrome 检索 |

## 五、通道选择优先级

```
1. 有 URL + 只需提取内容？ → WebFetch / Jina
2. 只需发现信息来源？ → WebSearch
3. 用户指向本人访问过的页面 / 组织内部系统？ → 本地 Chrome 检索
4. 需要登录 / 交互操作 / 已知反爬平台？ → CDP 浏览器
```

## 六、关键提示

- **优先使用 CDP**：小红书、微信公众号等公开内容也被反爬限制，直接 CDP
- **保留完整 URL**：站点内交互产生的链接包含必需参数，不要裁剪
- **GUI 优于程序化**：不确定时用 GUI 交互（点击、滚动），避免触发反爬
- **并行提速**：多个独立任务用子 Agent 并行，节省时间
- **一手来源**：核实信息时直访官网/原文，不依赖二手报道
