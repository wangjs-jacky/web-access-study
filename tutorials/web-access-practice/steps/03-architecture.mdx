# 架构全貌：四层调度 + CDP Proxy + 并行分治

## 设计哲学

不给 AI 固定操作流程，而是教它如何思考。SKILL.md 以"技术事实"形式呈现 tradeoff。

## 系统架构图

```
┌──────────────────────────────────────────────────┐
│                   AI Agent                        │
└───────────┬──────────────────────────────────────┘
            ▼
┌──────────────────────────────────────────────────┐
│              SKILL.md（思考框架）                  │
└───────────┬──────────────────────────────────────┘
    ┌───────┼───────────┐
    ▼       ▼           ▼
WebSearch  WebFetch    CDP Proxy (HTTP :3456)
                         │
                         ▼
                   Chrome (用户浏览器)
```

## 核心设计决策

| 问题 | 方案 | 原因 |
|------|------|------|
| 如何指导 AI？ | 目标导向 + 技术事实 | 让 AI 理解 tradeoff 后自主决策 |
| 如何连接浏览器？ | 直连用户日常 Chrome | 复用登录态，零配置 |
| 如何暴露 CDP？ | HTTP Proxy 转发 | Agent 只能执行 bash 命令 |
| 多任务处理？ | 并行分治 + tab 隔离 | 效率提升 + 保护主上下文 |
| 经验管理？ | Markdown 按域名存储 | 简单直观，易编辑和版本管理 |
